# 要件メモ

## 背景

現在はiPhoneのメモで支出を記録している。専用アプリにすることで、入力速度、一覧性、月別集計を改善したい。

家庭内の2人分の支出を分けて記録し、a と b のどちらがいくら多く払っているかをすぐ確認できるようにする。

## 最重要要件

- スマホで素早く入力できること
- 入力項目が多すぎないこと
- a と b の支出を分けて記録できること
- 今どちらが多く払っているか差額が分かること
- 後から月別・カテゴリ別に振り返れること
- オフラインでも使える方向にしやすいこと

## 現在のメモ運用

- 以前のメモでは `-` で記載している支出が a、`+` で記載している支出が b
- 例: `-48000-48000+7200+1200`
- アプリでは `-` / `+` は表示せず、a/b の選択ボタンで記録する
- a/b は内部IDとして扱い、画面上の表示名はあとから変更できる

## 支出データ

```ts
type Expense = {
  id: string;
  date: string;
  amount: number;
  category: string;
  person: "a" | "b";
  note?: string;
  createdAt: string;
  updatedAt: string;
};
```

## 初期カテゴリ案

- 食費
- 外食
- 日用品
- 交通
- 娯楽
- 医療
- サブスク
- その他

## 画面案

### 入力画面

- 金額入力を最優先で表示
- 支払った人を a/b から選択
- a/b の表示名を変更できる
- カテゴリはボタン選択
- メモは任意
- 保存後は入力欄をリセット

### 履歴画面

- 日付順の支出一覧
- 月別フィルタ
- 削除・編集

### 集計画面

- 今月の合計
- a の合計
- b の合計
- a/b の差額
- カテゴリ別合計
- 前月比較は後回し

## 判断メモ

- 最初はログインなし。
- 最初はlocalStorageで十分。
- データ移行を考えて、CSVエクスポートは早めに追加候補にする。
